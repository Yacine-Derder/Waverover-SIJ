4. COMPLETE SYNTHETIC RECORDED RUN
----------------------------------

run_experiment starts rosbag, the synthetic MCS publisher, and the coordinator.
It forces the coordinator into dry-run and therefore records computed results
without publishing rover waypoints.

Centralized heuristic:

ros2 run waverover_swarm_controller run_experiment   --config /home/derder/waverover_experiments/dynamic_heuristic.yaml

Decentralized heuristic:

ros2 run waverover_swarm_controller run_experiment   --config /home/derder/waverover_experiments/dynamic_heuristic_decentralized.yaml

Centralized convex:

ros2 run waverover_swarm_controller run_experiment   --config /home/derder/waverover_experiments/dynamic_convex.yaml

Centralized MPC:

ros2 run waverover_swarm_controller run_experiment   --config /home/derder/waverover_experiments/dynamic_mpc_centralized.yaml

Distributed MPC:

ros2 run waverover_swarm_controller run_experiment   --config /home/derder/waverover_experiments/dynamic_mpc_distributed.yaml

Stop a run cleanly with Ctrl-C. Do not use kill -9.

For independent random synthetic motion, the YAML must contain:

synthetic_mcs:
  mode: random_walk
  seed: 2026
  duration_sec: 120.0
  formation_coupling: independent
  initial_radius_m: 1.0
  connectivity_policy: observe

Use seed: null for a different non-reproducible seed on each run.


5. STANDALONE FAKE-MCS DRY-RUN
------------------------------

Terminal 1 - synthetic poses:

ros2 launch waverover_swarm_controller synthetic_mcs.launch.py   config_file:=/home/derder/waverover_experiments/dynamic_heuristic.yaml   rate_hz:=20.0

Terminal 2 - coordinator, no waypoint publication:

ros2 launch waverover_swarm_controller swarm_controller.launch.py   config_file:=/home/derder/waverover_experiments/dynamic_heuristic.yaml   algorithm:=heuristic   dry_run:=true

Terminal 3 - verify pose rate:

ros2 topic hz /macortex_bridge/waverover_131/pose

Terminal 3 - inspect generated results:

ros2 topic echo /waverover_swarm/diagnostics

or:

ros2 topic echo /waverover_swarm/controller_telemetry

Dry-run computes, validates, and records results but publishes nothing on
/waverover_<ID>/waypoints or /waverover_<ID>/end_trial.


6. FAKE-MCS WAYPOINT-PUBLICATION TEST
-------------------------------------

Keep the standalone synthetic MCS publisher running.

Terminal 3 - start listening before live launch:

ros2 topic echo /waverover_131/waypoints

Terminal 2 - relaunch the coordinator (automatic publication):

ros2 launch waverover_swarm_controller swarm_controller.launch.py   config_file:=/home/derder/waverover_experiments/dynamic_heuristic.yaml   algorithm:=heuristic   dry_run:=false

With fresh synchronized poses and a valid result, one active waypoint is
published automatically per rover and refreshed every 1 second. Pending
setpoints continue to be computed, but a new waypoint is not handed off until
the rover echoes its command stamp on `/waverover_<ID>/waypoint_reached`.
Listen with `ros2 topic echo /waverover_131/waypoint_reached`. After 10 seconds the active
waypoint is reported overdue without faulting or stopping refresh. The onboard
controller coalesces identical refresh messages. Stop with Ctrl-C for graceful
end_trial cleanup.


7. REAL MCS WITH ROVERS TURNED OFF
----------------------------------

Do not run synthetic_mcs while the real MCS bridge is publishing. Both sources
use the same topics and must never run simultaneously.

Verify discovered topics:

ros2 topic list | sort

Verify each rover pose, replacing 131 with IDs 132 through 136:

ros2 topic info /macortex_bridge/waverover_131/pose --verbose
ros2 topic echo /macortex_bridge/waverover_131/pose --once
ros2 topic hz /macortex_bridge/waverover_131/pose

Every pose must use frame_id robotics_lab, finite coordinates, and a valid
quaternion. Fresh synchronized poses are required for every robot_ids entry.

Start with a real-MCS dry-run:

ros2 launch waverover_swarm_controller swarm_controller.launch.py   config_file:=/home/derder/waverover_experiments/dynamic_heuristic.yaml   algorithm:=heuristic   dry_run:=true

Inspect all computed setpoints:

ros2 topic echo /waverover_swarm/controller_telemetry

To test actual waypoint publication with the rovers off, stop the dry-run
coordinator, start the listener first:

ros2 topic echo /waverover_131/waypoints

Then relaunch in live mode; validated results dispatch automatically:

ros2 launch waverover_swarm_controller swarm_controller.launch.py   config_file:=/home/derder/waverover_experiments/dynamic_heuristic.yaml   algorithm:=heuristic   dry_run:=false

Stop the coordinator with Ctrl-C when finished and verify end_trial for each
commanded rover.


8. RECORD REAL MCS WITHOUT WAYPOINT PUBLICATION
-----------------------------------------------

Set these fields in the selected YAML:

recording:
  root_directory: /home/derder/waverover_runs
  profile: core
  storage_id: sqlite3
  pose_source: mcs
  start_synthetic: false

Then use the normal supervisor command:

ros2 run waverover_swarm_controller run_experiment   --config /home/derder/waverover_experiments/dynamic_heuristic.yaml

The supervisor respects safety.dry_run from this YAML. Keep it true for a
non-publishing recording; false enables automatic validated waypoint dispatch.

12. IMPORTANT BEHAVIOR
----------------------

- run_experiment respects safety.dry_run from the selected experiment YAML.
- The synthetic trajectories are open-loop and ignore controller waypoints.
- A coordinator with dry_run:=false automatically publishes after fresh
  synchronized poses and a valid result; start listeners before launching it.
- A coordinator fault after dispatch begins is latched until process restart.
- Active waypoints refresh at waypoint_dispatch.refresh_period_sec; the overdue
  threshold is warning-only and does not publish end_trial.
- Fake and real MCS publishers must not run at the same time.
- The algorithm launch argument overrides controller.algorithm in the YAML.
- Configuration changes do not require rebuilding the ROS workspace.
- Use Ctrl-C for graceful cleanup and rosbag finalization.
